# MonitorAccounts 页面设计

## 路由与权限

- 路由：`/monitor/accounts`
- 权限：`ADMIN`

## 页面目标

- 监控账号池健康度、吞吐、错误率和分布。

## 核心模块

- 汇总卡片（活跃、异常、消耗）
- Token 趋势图
- 渠道占比图
- 账号健康明细表

## 接口契约

- 当前前端调用：`GET /api/admin/monitor/accounts?timeRange=...`
- 需与后端统一契约并明确字段。

## 关键交互

- 时间范围切换
- 自动刷新与手动刷新

## 状态与异常

- 数据为空时显示空表与空图。
- 错误时显示可重试入口。

## 验收标准

- 可定位到具体账号的健康状态与异常原因。
- 与账号池管理页形成运维闭环。
