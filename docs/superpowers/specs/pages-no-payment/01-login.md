# Login 页面设计

## 路由与权限

- 路由：`/login`
- 权限：未登录可访问；已登录自动跳转。

## 页面目标

- 完成账号登录并建立会话。

## 核心模块

- 用户名输入
- 密码输入与显隐
- 记住用户名
- 提交按钮
- 错误提示
- 去注册入口

## 接口契约

- `POST /api/auth/login`
- 入参：`username`、`password`
- 出参：用户会话信息（含角色）

## 关键交互

- 前端校验必填项。
- 成功后按角色跳转：
- `ADMIN -> /`
- `USER -> /my-api-keys`

## 状态与异常

- 提交中按钮禁用。
- 登录失败显示后端 message。
- 网络异常显示通用错误文案。

## 验收标准

- 能正确写入会话并跳转目标页。
- 刷新后保持登录状态。
