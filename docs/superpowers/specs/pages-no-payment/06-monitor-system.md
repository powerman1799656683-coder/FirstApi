# MonitorSystem 页面设计

## 路由与权限

- 路由：`/monitor/system`
- 权限：`ADMIN`

## 页面目标

- 监控系统级资源与运行稳定性。

## 核心模块

- CPU/内存/JVM/DB 指标卡
- 网络与节点信息
- 系统告警列表
- 时间范围筛选与手动刷新

## 接口契约

- 当前前端调用：`GET /api/admin/monitor/system?timeRange=...`
- 需与后端统一：建议明确 `system` 聚合接口，或前端改走统一 `/api/admin/monitor` 数据模型。

## 关键交互

- 轮询刷新（短周期）
- 时间范围切换后立即重新加载

## 状态与异常

- 不依赖 mock 作为主路径。
- 接口失败时显示降级提示，不静默误导。

## 验收标准

- 指标与告警均可追溯到真实后端数据。
- 页面在高频刷新下保持可用。
