# Register 页面设计

## 路由与权限

- 路由：`/register`
- 权限：未登录可访问；已登录不可访问。

## 页面目标

- 创建普通用户并自动登录。

## 核心模块

- 用户名输入
- 密码输入
- 确认密码输入
- 提交按钮
- 错误提示
- 去登录入口

## 接口契约

- `POST /api/auth/register`
- 入参：`username`、`password`、`confirmPassword`
- 出参：用户会话信息

## 关键交互

- 前端校验必填与两次密码一致。
- 注册成功跳转 `/my-api-keys`。

## 状态与异常

- 提交中按钮禁用。
- 注册失败显示后端 message。

## 验收标准

- 注册成功后无需再次登录。
- 用户角色为普通用户并进入用户首页。
