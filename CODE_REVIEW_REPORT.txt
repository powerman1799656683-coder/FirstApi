【Bug #1】ClaudeRelayAdapter NPE 风险 (构造函数设计缺陷)

位置: ClaudeRelayAdapter.java 第 30-45 行

问题分析:
  - 第 30-32 行存在非 @Autowired 构造函数:
    public ClaudeRelayAdapter(ObjectMapper objectMapper, RelayProperties relayProperties) {
        this(objectMapper, relayProperties, null, null, null);  ← 全部设为 null!
    }
  
  - 这会导致: relayAccountSelector = null
  
  - 第 216 行调用时发生 NPE:
    String baseUrl = relayAccountSelector.resolveBaseUrl(...)

影响: 账号测试功能完全不可用

修复建议:
  方案 A: 删除第一个构造函数 (推荐)
  方案 B: 添加 null check 防御


【Bug #2】OpenAI OAuth stream 参数完全相反

位置: OpenAiRelayAdapter.java 第 196 行 vs 第 275 行

问题分析:
  - probeAccount() (第 196): reqBody.put("stream", false);
  - relayResponses() (第 275): node.put("stream", true);
  
  参数完全相反!

影响: 
  - OAuth 账号测试时用 stream=false (关闭流式)
  - 实际中转用 stream=true (流式响应)
  - 测试通过但实际中转失败的风险

修复: 
  将第 196 行改为:
  reqBody.put("stream", true);

【Bug #5】isFatalError() 逻辑缺陷 (操作符优先级)

位置: AccountService.java 第 503-506 行

代码:
  if ("invalid_api_key".equals(c) || "account_deactivated".equals(c)
          || "insufficient_quota".equals(c) && statusCode == 403) {
      return true;
  }

问题:
  由于 && 优先级高于 ||，实际为:
  (invalid_api_key) || (account_deactivated) || ((insufficient_quota) && (statusCode==403))
  
  不一致处:
  - invalid_api_key: 总是返回 true (不管状态码)
  - account_deactivated: 总是返回 true (不管状态码)
  - insufficient_quota: 只有 403 时才返回 true
  
修复: 添加注释说明意图，或调整逻辑
