# FirstApi 订阅与扣费（Billing）系统 - 完整分析

## 📊 系统架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                    用户 API 请求                                │
└───────────────────┬─────────────────────────────────────────────┘
                    │
                    ↓
        ┌───────────────────────┐
        │   RelayService        │  ← 中转路由服务
        │  - 账户选择            │
        │  - 请求转发            │
        └───────────┬───────────┘
                    │
                    ↓
        ┌───────────────────────┐
        │  后端 API 响应         │
        │  - token 数            │
        │  - 状态码              │
        └───────────┬───────────┘
                    │
                    ↓
     ┌──────────────────────────────────┐
     │  RelayRecordService.record()     │  ← 核心扣费处理
     │  ├─ 计算成本                     │
     │  ├─ 判断优先级                   │
     │  ├─ 执行扣费                     │
     │  └─ 记录用量                     │
     └──────────────┬───────────────────┘
                    │
         ┌──────────┴──────────┐
         │                     │
         ↓                     ↓
    ┌─────────────┐      ┌──────────────┐
    │ 有活跃订阅？ │      │ 记录每日用量 │
    └─────┬───────┘      └──────┬───────┘
          │                     │
      ┌───┴──────┐              │
      │           │              │
      ↓           ↓              │
   [是]        [否]              │
      │           │              │
      ↓           ↓              ↓
   扣订阅   扣用户余额   daily_quota_usage
    额度      (balance)      表 +cost
